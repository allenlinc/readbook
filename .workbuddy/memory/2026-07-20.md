### Follow-up 2: "both languages still show" + "no English explanation" → ROOT CAUSE = asset caching
- User reported中英同时显示 + 英文无解释 even after the mode rework. Confirmed live CSS/JS already had the new `lang-en` rules and `explain.en` data is clean (0 CJK in explain.en across all 72). Root cause = **GitHub Pages/browser cached the OLD `style.css`+`quiz.js`**; new `index.html` (with `lang-en` body) loaded but old CSS lacked the hide rule → both showed, old JS had different structure.
- Fix: **cache-bust** — appended `?v=2` to `assets/style.css` and `assets/quiz.js` references in build.py (hub link + book link/js). New URL → browser fetches fresh.
- Also plugged bare-Chinese leaks so EN mode is 100% English: hub/JS `card()` "开始 ▶" → wrapped `<span class="en">Start ▶</span><span class="zh">开始 ▶</span>`; book hero tag "10 Q · EN / 中文" → `bi("10 Q","10 题")`; hub search placeholder set via JS (`setPH()`) per mode; empty-state "没有找到匹配的书籍 No matches." → `bi(...)`.
- quiz.js: added defensive default — if body has neither `lang-en` nor `lang-zh`, force `lang-en` (guards stale HTML).
- Rebuilt, pushed (`d337c96`), Pages rebuild queued. Verified live: hub/book HTML reference `?v=2`, CSS has `lang-en .zh{display:none!important}`, book page 200.
- Lesson: after editing shared `assets/*.css|*.js` for a deployed static site, bump a version query on the asset URLs (or tell user hard-refresh) — otherwise stale cached assets desync from new HTML and produce exactly this "looks unchanged / broken" symptom.

### Follow-up 3: hero skill tag was white-on-white (invisible)
- User: EN-mode "Key Ideas and Details" skill label looked white/invisible. Root cause: hero `.tag` used `background:rgba(255,255,255,.16)` with NO color → inherited hero `color:#fff` → white text on a 16%-white pill over the teal gradient = near-zero contrast (esp. long labels). The feedback `.skill` (blue on #eef2ff) was fine; only the hero `.tag` was the culprit.
- Fix: `.tag` → `background:rgba(255,255,255,.92);border:1px solid rgba(255,255,255,.7);color:var(--primary-d)` (white pill, dark-teal text = high contrast). Bumped asset version `?v=2`→`?v=3` in build.py so browsers refetch the new CSS. Rebuilt, pushed (`b017b07`), Pages rebuild triggered; live `.tag` rule + `?v=3` refs verified.
